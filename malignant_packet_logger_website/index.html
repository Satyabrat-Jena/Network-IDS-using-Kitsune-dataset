Overview

The Malignant Packet Logger is a system designed to securely log features of malignant network packets to the Ethereum blockchain. This provides an immutable, transparent record of network security incidents that can be verified and analyzed.

Key Components
  • Smart Contract: Ethereum-based storage for packet features
  • Web3.py Interface: Python library for interacting with the blockchain
  • Feature Extraction: Selecting and preparing packet data for storage
  • Verification Mechanism: Allowing trusted authorities to verify incidents

The system is designed to work with the Kitsune Intrusion Detection System (IDS) dataset, which contains network packet data with numerous numerical features. Each packet is represented as a row in the dataset, with columns representing different features.

Smart Contract

The MalignantPacketLogger smart contract is written in Solidity and deployed on the Ethereum blockchain. It provides functions for logging packet features, retrieving packet information, and verifying packets.

Contract Structure
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract PacketLogger {
    struct Packet {
        address sender;
        uint256 timestamp;
        string features;
    }

    Packet[] public packets;

    event PacketLogged(address indexed sender, uint256 timestamp, string features);

    function logPacket(string memory _features) public {
        packets.push(Packet(msg.sender, block.timestamp, _features));
        emit PacketLogged(msg.sender, block.timestamp, _features);
    }

    function getPacketCount() public view returns (uint256) {
        return packets.length;
    }

    function getPacket(uint256 index) public view returns (address, uint256, string memory) {
        require(index < packets.length, "Index out of bounds");
        Packet memory p = packets[index];
        return (p.sender, p.timestamp, p.features);
    }
}
}

Create a file named deploy_contract.py with the deployment script and run it:

Web3.py Integration

Web3.py is used to interact with the Ethereum blockchain and the deployed smart contract. Here's how to integrate it into your application:

Connecting to the Contract
import json
from web3 import Web3

# Load contract details
def load_contract():
    # Load ABI
    with open('contract_abi.json', 'r') as f:
        contract_abi = json.load(f)
    
    # Load contract address
    with open('contract_address.txt', 'r') as f:
        contract_address = f.read().strip()
    
    # Connect to Ethereum node
    w3 = Web3(Web3.HTTPProvider('http://127.0.0.1:8545'))
    
    # Check connection
    if not w3.is_connected():
        raise Exception("Failed to connect to Ethereum node")
    
    # Get account
    account = w3.eth.accounts[0]
    
    # Create contract instance
    contract = w3.eth.contract(address=contract_address, abi=contract_abi)
    
    return w3, contract, account
Logging a Packet
def log_malignant_packet(contract, account, packet_data, attack_type):
    """
    Log a malignant packet to the blockchain.
    
    Args:
        contract: Web3 contract instance
        account: Ethereum account to use for transaction
        packet_data: Packet features (pandas Series or dict)
        attack_type: String describing the attack type
    
    Returns:
        transaction_receipt: Receipt of the blockchain transaction
    """
    # Extract important features
    feature_indices, feature_values = extract_features(packet_data)
    
    # Log the packet
    tx = contract.functions.logPacket(
        attack_type,
        feature_indices,
        feature_values
    ).build_transaction({
        'from': account,
        'gas': 3000000,
        'gasPrice': Web3(Web3.HTTPProvider('http://127.0.0.1:8545')).eth.gas_price,
        'nonce': Web3(Web3.HTTPProvider('http://127.0.0.1:8545')).eth.get_transaction_count(account)
    })
    
    # Send the transaction
    tx_hash = Web3(Web3.HTTPProvider('http://127.0.0.1:8545')).eth.send_transaction(tx)
    
    # Wait for the transaction to be mined
    tx_receipt = Web3(Web3.HTTPProvider('http://127.0.0.1:8545')).eth.wait_for_transaction_receipt(tx_hash)
    
    return tx_receipt

Feature Selection

Since storing all 100+ features would be prohibitively expensive in terms of gas costs, it's important to select only the most relevant features for blockchain storage.

Feature Extraction Function
def extract_features(packet_data, important_feature_indices=None):
    """
    Extract the most important features from a packet.
    
    Args:
        packet_data: A pandas Series or dict containing packet features
        important_feature_indices: List of indices of important features to extract
                                  If None, selects the first 10 non-zero features
    
    Returns:
        feature_indices: List of feature indices
        feature_values: List of feature values (scaled for blockchain storage)
    """
    # If no specific indices provided, select first 10 non-zero features
    if important_feature_indices is None:
        # Convert to dict if Series
        if isinstance(packet_data, pd.Series):
            packet_dict = packet_data.to_dict()
        else:
            packet_dict = packet_data
            
        # Filter out non-numeric columns and the label column 'x'
        numeric_features = {k: v for k, v in packet_dict.items() 
                           if k != 'x' and isinstance(v, (int, float))}
        
        # Sort by absolute value to find most significant features
        sorted_features = sorted(numeric_features.items(), 
                                key=lambda x: abs(x[1]), 
                                reverse=True)
        
        # Take top 10 or fewer if not enough features
        top_features = sorted_features[:min(10, len(sorted_features))]
        
        # Extract indices and values
        feature_indices = [int(k) if isinstance(k, (int, str)) and k.isdigit() else i 
                          for i, (k, _) in enumerate(top_features)]
        feature_values = [v for _, v in top_features]
    else:
        # Use the provided indices
        feature_indices = important_feature_indices
        feature_values = [packet_data[idx] for idx in important_feature_indices]
    
    # Scale values for blockchain storage (multiply by 10^6 to preserve decimals)
    scaled_values = [int(v * 1_000_000) for v in feature_values]
    
    return feature_indices, scaled_values

Feature Selection Strategies

Magnitude-Based Selection

Select features with the largest absolute values, as these often represent the most significant anomalies.

Domain Knowledge

Select features known to be important for specific attack types based on network security expertise.

Gas Optimization

Gas optimization is crucial for cost-effective blockchain logging. Here are strategies to minimize gas costs:

Optimization Strategies
  • Feature Limitation: Store only 10-20 key features instead of all 100+
  • Batch Processing: Log multiple packets in a single transaction when possible
  • Data Scaling: Scale floating-point values to integers to avoid complex storage
  • Efficient Data Structures: Use arrays instead of mappings when appropriate
  • Off-chain Storage: Consider storing full feature sets off-chain with only hashes on-chain
Gas Usage Example

Logging a packet with 10 features typically consumes approximately 500,000-600,000 gas. At a gas price of 20 Gwei, this would cost about 0.01-0.012 ETH per transaction.

About the Project

Project Background

The Malignant Packet Logger project was developed to address the need for secure, immutable records of network security incidents. By leveraging blockchain technology, this system provides a tamper-proof method for logging features of malignant network packets detected by intrusion detection systems.

This approach ensures that security incidents cannot be altered or deleted, creating a reliable audit trail that can be used for forensic analysis, compliance reporting, and security research.

Kitsune IDS Integration

This project is designed to work with the Kitsune Intrusion Detection System (IDS), a machine learning-based network anomaly detection system. Kitsune IDS analyzes network traffic and identifies malignant packets based on various features.

The Malignant Packet Logger extends Kitsune's capabilities by providing a secure storage mechanism for the features of detected malignant packets, ensuring that evidence of security incidents is preserved in an immutable form.

The dataset used for this project contains various attack types including ARP MitM, Active Wiretap, Fuzzing, Mirai, OS Scan, SSDP Flood, SSL Renegotiation, SYN DoS, and Video Injection.
